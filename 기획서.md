# 모바일 청첩장 템플릿 기반 생성기 기획서

## 1. 화면 구성

### 1-1. 메인 페이지
- 생성된 청첩장 목록을 나열 (청첩장 이름, 배포 URL, URL 복사 버튼)
- "새 청첩장 만들기" 버튼으로 편집기 진입
- 로그인 없이 누구나 접속하여 청첩장 생성 및 배포 URL 생성 가능

### 1-2. 편집기 페이지
- 좌우 분할 레이아웃 구성
  - **좌측 패널 (편집 영역)**: 컴포넌트 목록이 표시되며, `+` / `-` 버튼으로 컴포넌트를 추가/제거할 수 있다.
  - **우측 패널 (미리보기 영역)**: 좌측에서 구성한 컴포넌트 배열이 반영된 청첩장의 렌더링된 웹뷰가 실시간으로 표시된다.
- 모든 컴포넌트는 추가/삭제/중복 사용 가능
- **기본 컴포넌트**: 새 청첩장 생성 시 자동으로 등록되어 있는 컴포넌트 (삭제 가능)
  - 인사말, 신랑 정보, 신부 정보, 예식 일시, 예식 장소, 연락처
  - 상세 정의: [컴포넌트_정의.md](컴포넌트_정의.md) 참조
- 컴포넌트 순서 변경: 드래그 앤 드롭으로 컴포넌트 배치 순서를 변경할 수 있다.
- 컴포넌트 종류 및 세부 정의: [컴포넌트_정의.md](컴포넌트_정의.md) 참조

### 1-3. 청첩장 열람 페이지
- URL: `mwi.onethelab.com/{배포명}`
- 편집기에서 저장된 JSON 데이터를 기반으로 청첩장을 렌더링
- 같은 앱 내에서 라우팅으로 처리

## 2. 컴포넌트 추가/삭제

- **추가 (`+` 버튼)**: 모든 컴포넌트 이름이 표시되는 모달에서 선택. 선택 후 모달이 닫히며 가장 마지막 위치에 컴포넌트가 추가된다.
- **삭제 (`-` 버튼)**: 해당 컴포넌트를 목록에서 제거. 삭제된 컴포넌트의 입력 데이터는 보관되어, 동일 컴포넌트를 다시 추가 시 이전 입력값이 복원된다.
- 같은 종류의 컴포넌트를 중복으로 추가할 수 있다.

## 3. 편집 모드

- 각 컴포넌트별로 **Basic / Advanced** 토글 제공
  - **Basic 모드**: 핵심 콘텐츠만 입력 (예: 인사말 문구, 주소, 혼주 이름 등). 스타일은 자동 적용된다.
  - **Advanced 모드**: Basic 항목에 더해 세부 스타일을 조절할 수 있다.
    - 글씨 크기, 폰트 종류
    - 내부 여백(padding), 바깥 여백(margin)
    - 가로 너비, 세로 너비
    - 내부 항목이 복수인 경우 항목 간 간격(gap) 조정
  - 각 컴포넌트별 Basic/Advanced 항목은 [컴포넌트_정의.md](컴포넌트_정의.md)에 정의 예정
- 모든 입력란에 **복사/붙여넣기 버튼** 제공하여 컴포넌트 간 내용을 쉽게 복붙할 수 있도록 구성

## 4. 실시간 미리보기

- 좌측 편집 영역에서 컴포넌트 추가/삭제/순서 변경/내용 수정 시 우측 웹뷰에 실시간 반영
- 청첩장 규모(컴포넌트 10~20개 수준)에서는 DOM 갱신 비용이 낮아 성능 문제 가능성이 낮음
- 반응형 웹: 데스크톱, 모바일 모두 대응. 모바일에서는 확대/축소 없이 세로 스크롤로 열람 가능

### 주의 사항 및 캐싱 전략

- **이미지 컴포넌트**: 고해상도 이미지를 매 렌더링마다 재로드하지 않도록 이미지 캐싱 및 썸네일 활용 필요
- **지도 컴포넌트**: 외부 API(카카오맵, 네이버맵 등) 호출 시 컴포넌트 순서 변경마다 지도를 재생성하지 않도록 지도 인스턴스 캐싱 필요
- **웹뷰 구현 방식**: iframe 사용 시 전체 페이지 재로드 대신, 같은 SPA 내 상태 공유 또는 postMessage로 변경분만 전달하는 방식 권장

## 5. 데이터 구조

- 청첩장의 모든 정보 및 설정값은 **JSON 파일**로 저장
- 청첩장 렌더러가 JSON 값을 읽어 청첩장을 렌더링하는 구조
- 컴포넌트 삭제 후 재추가 시 이전 입력값 복원 (JSON 내 보관)

### JSON 구조 예시
```json
{
  "id": "honggildong2026",
  "createdAt": "2026-03-07",
  "components": [
    {
      "type": "greeting",
      "order": 0,
      "basic": {
        "title": "소중한 분들을 초대합니다",
        "content": "서로 다른 길을 걸어온 두 사람이..."
      },
      "advanced": {
        "fontFamily": "Noto Serif KR",
        "textAlign": "center",
        "backgroundColor": "#ffffff"
      }
    },
    {
      "type": "groomInfo",
      "order": 1,
      "basic": {
        "name": "홍길동",
        "fatherName": "홍판서",
        "motherName": "춘섬"
      },
      "advanced": {
        "relation": "장남",
        "deceasedMark": { "father": false, "mother": false }
      }
    }
  ],
  "deletedComponents": []
}
```

## 6. 기술 스택

- **프론트엔드**: React (필요 시 추가 라이브러리 사용 가능)
- **백엔드**: 이미지 업로드/저장 API (Node.js/Express 등)
- **데이터 저장**: JSON 파일 (서버 파일시스템)
- **이미지 저장**: Vultr 서버 내 파일시스템에 보관
- **배포 환경**: Vultr Ubuntu 서버

## 7. 배포

- **도메인**: `mwi.onethelab.com`
- **청첩장 URL**: `mwi.onethelab.com/{배포명}`
  - 배포명: 사용자가 청첩장 생성 시 직접 입력
  - 규칙: 영문 + 숫자, 6자 이상, 기존 배포명과 중복 불가
- React 빌드 후 Vultr 서버에서 정적 파일 서빙
- 같은 앱 내 라우팅으로 편집기/열람 페이지 처리

## 8. 향후 계획 (미구현)

- **프리셋**: 컴포넌트 조합을 미리 구성한 세트
- **테마**: 색상, 레이아웃, 여백 등이 조정된 스타일 설정값
- **템플릿**: 프리셋 + 테마의 조합으로 제공
- 편집기 완성 후 순차적으로 추가 예정
